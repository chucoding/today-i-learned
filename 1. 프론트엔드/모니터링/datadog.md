# Datadog

## 사용 이유
프론트엔드 성능 개선과 비즈니스 인사이트 확보 가능

- 성능 최적화 실시간 웹 바이탈
- 문제 해결 오류 및 로그 추적
- 비즈니스 인사이트 사용자 행동 분석
- 비용 효율성 리소스 사용 최적화

💡 프론트엔드 개발자가 비즈니스 로직을 이해하고 데이터를 분석하는 역량을 키우는 것이 중요

## 모니터링 종류
- 성능 관점 : 애플리케이션 작동 상태, 엔드포인트별 상태, 사용자 불편 구간
- 트러블슈팅 관점 : 에러 발생시 영향받는 사용자 파악, 에러 빈도 및 추이 분석, 사용자별 에러 발생 여정 분석
- 비즈니스 관점 : 특정 기능 사용 현황 통계, 지역별/기능별 사용 현황, 장애 발생 시 매출 손실 연계

## 1. 성능 모니터링
코어 웹 바이탈(Core Web Vitals)을 통해 사용자 관점의 성능을 직관적으로 측정

### 종류
- LCP (Largest Contentful Paint) : 콘텐츠 중 가장 큰 요소가 로딩되는 시간 측정.
- FCP (First Contentful Paint) : 처음으로 콘텐츠가 화면에 그려지는 시간 측정.
- CLS (Cumulative Layout Shift) : 레이아웃 변경으로 인한 시각적 불안정성 측정.
- FID (First Input Delay) : 사용자의 첫 입력에 대한 응답 시간 측정.
- INP (Interaction to Next Paint) : 사용자의 모든 상호작용에 대한 응답 시간 측정.

### 기능
- 홈페이지 로딩 지연 파악
- 리소스별 로딩 시간 분석을 통해 병목 구간 찾기
- 프론트부터 백엔드까지 E2E 모니터링을 통해 성능 문제의 근본 원인 추적
- 페이지뷰, 총 에러 수, 버전별 성능 지표 확인
- 사용자 디바이스, 브라우저, 지역별 성능 데이터 제공
- 뷰별 세션 수, 소요 시간 등 확인

## 2. 트러블슈팅
### 2.1. 에러 트래킹
발생하는 수 많은 에러를 우선순위에 따라 분류하고 신속하게 해결

### 2.2. 세션 리플레이
> 데이터독 고유 기능

실제 사용자가 서비스를 이용하는 여정을 시간 순서대로 재현하여 문제 발생 지점을 파악
> 저장 기간: 기본 30일

### 2.3. E2E 분석
> 데이터독 고유 기능

프론트엔드에서 발생한 이슈부터 백엔드 API, 호스트, 컨테이너, DB 통신까지 연계하여 분석

## 3. 비즈니스 인사이트
### 3.1. 프로덕트 애널리틱스
별도의 설정 없이도 기본적으로 제공되는 BI 대시보드를 통해 비즈니스 인사이트를 얻을 수 있음.
- 주요 지표: 액티브 유저, 페이지뷰, 지속 시간, 이탈률 등 확인 가능
- 사용자 분석: MAU, DAU 확인, 지역별/도시별 사용자 분포 파악 가능
- 인기 페이지 분석: 어떤 페이지가 인기가 많은지 확인 가능

### 3.2. 패스웨이(Pathways) 기능
사용자 여정의 특정 구간(예: 장바구니)을 기준으로 인기 제품이나 인기 아이템을 자동으로 파악.

### 3.3. 히트맵
사용자들이 페이지에서 어떤 영역을 클릭하고 어디까지 스크롤 하는지 시각적으로 보여줌.
