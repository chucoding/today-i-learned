# 라우팅
요즘은 프론트에서 페이지 라우팅을 관리하기 때문에 체계적인 설계가 필요함.

## 라우팅이 필요없는 경우
- 간단한 화면, 모바일 앱 등

## 라우팅이 필요한 경우
- “다시 들어왔을 때” 같은 화면을 복원
  - 라우팅이 없으면 보통 localStorage/IndexedDB에 상태를 저장해서 복원해야 하는데, 설계가 더 복잡해질 수 있음.
- “딥링크/공유 링크”가 가능해짐
- SEO가 아니라도 “북마크/히스토리”가 유용함
  - 라우팅이 없으면 뒤로가기가 “앱 내 화면 이동”이 아니라 “앱 종료/이전 사이트”가 되어 UX가 깨질 수 있음.
- 화면이 커질수록 상태 분리가 필요
  - view = 'home' | 'detail' | ... 같은 자체 상태 머신을 만들게 되고 점점 라우터를 “직접 구현”하게 되는 경우가 많아.
