# 타입스크립트 문법
기본적으로 다음과 같이 선언한다.
```tsx
let 변수명: 타입 = 값;
```

# Boolean
```tsx
let isBoolean: boolean;
let isDone: boolean = false;
```

# Number
```tsx
let num: number;
let integer: number = 6;
let float: number = 3.14;
```

# String
```tsx
let str: string = 'hello world!';
let myColor: string `My color is ${red}.`;
```

# Array
```tsx
// 문자열만 가지는 배열
let fruits: string[] = ['Apple', 'Banana', 'Mango'];
let fruits: Array<string> = ['Apple', 'Banana', 'Mango'];

// 숫자만 가지는 배열
let oneToSeven: number[] = [1,2,3,4,5,6,7];
let oneToSeven: Array<number> = [1,2,3,4,5,6,7];

//유니언 타입
let array: (string|number)[] = ['Apple',1,2,'Banana'];
let array: Array<string|number> = ['Apple',1,2,'Banana'];

//any 타입
let someArr: any[] = [0, 1, {}, [], 'str', false];

//readonly 배열
let arrA: readonly number[] = [1, 2, 3, 4];
let arrB: readonlyArray<number> = [0, 9, 8, 7];
```

# Interface
```tsx
interface IUser {
	name: string,
	age: number,
	isValid?: boolean // Optional property
}
let userArr: IUser[] = [
	{
		name: 'Neo',
		age: 85
	},
	{
		name: 'Lewis',
		age: 52,
		isValid: false
	}
];
```

> 💡 **interface vs type**
>
> - **interface**: 객체 형태의 타입을 선언할 때 주로 사용. 같은 이름으로 **선언 병합(declaration merging)**이 가능해서, 같은 인터페이스를 여러 번 선언하면 자동으로 합쳐진다.
> - **type**: 객체뿐 아니라 유니언(`|`), 인터섹션(`&`), 원시/리터럴 타입 등 **다양한 타입 표현**이 가능하다.
> - 객체 타입만 정의할 때는 둘 다 쓸 수 있지만, **확장**은 `interface`는 `extends`, `type`은 `&`(인터섹션)로 한다.
> - 공식 스타일 가이드에서는 **일관성**을 위해 객체는 `interface`, 유니언/매핑 등이 필요할 때만 `type`을 쓰는 방식을 권장한다.

```tsx
//read only 설정하는법
let user: Readonly<IUser> = {
	name: 'Neo',
	age: 36
};

//함수형 인터페이스
interface IGetUser {
	(name:string): String
}
```

# Tuple

```tsx
tuple = [1, 'a'] // Error - TS2322

//이런식으로 값도 사용가능
let tuple ; [1, number];
```

# Enum
```tsx
enum Week {
	Sun,
	Mon,
	Tue = 20,
	Wed,
}

console.log(Week.Mon); // 1
console.log(Week.Wed); // 21 수동으로 값을 변경할 수 있으며, 값을 변경한 부분부터 다시 1 씩 증가
console.log(Week[0]); // Sun  역방향 매핑(Reverse Mapping)
```

# Object
반복사용되는 변수인 경우 interface 또는 type을 사용하는것이 좋음
```tsx
let obj: object = {};
let userA: { name: string, age: number } {
	name: 'HEROPY',
	age: 123
};

let userB: { name: string, age: number } {
	name: 'HEROPY',
	age: false, // Error
	email: 'thesecon@gmail.com' // Error
}
```
