# 패키지 구조

디자인시스템 구축시 @types/react, react, react-dom은 peerDependencies로만 관리

```json
{
  "name": "@rsquare/ds",
  "version": "1.0.0",
  "main": "dist/index.js",
  "types": "dist/index.d.ts",
  "files": [
    "dist"
  ],
  "scripts": {
    "dev": "vite build --watch",
    "build": "vite build",
    "clean": "rimraf dist",
    "type-check": "tsc --noEmit"
  },
  "dependencies": {
    "@vanilla-extract/css": "^1.17.4",
    "@vanilla-extract/recipes": "^0.5.7",
    "@vanilla-extract/sprinkles": "^1.6.5"
  },
  "devDependencies": {
    "@vanilla-extract/vite-plugin": "^5.1.1",
    "rimraf": "^6.0.1",
    "typescript": "^5.9.2",
    "vite": "^7.1.3"
  },
  "peerDependencies": {
    "react": ">=18.0.0",
    "react-dom": ">=18.0.0"
  }
}
```

# 프로젝트 구조

vanilla-extract로 디자인 시스템을 구축할 때는 **디자인 토큰 → 유틸리티 스타일 → 컴포넌트 스타일**로 점진적으로 쌓아가는 구조가 깔

```
src/
 ├─ styles/                     # 디자인 시스템 핵심
 │   ├─ tokens/                 # 디자인 토큰 (color, spacing, typography 등)
 │   │   ├─ color.css.ts
 │   │   ├─ spacing.css.ts
 │   │   ├─ typography.css.ts
 │   │   └─ index.ts
 │   │
 │   ├─ themes/                 # 테마 (light, dark 등)
 │   │   ├─ lightTheme.css.ts
 │   │   ├─ darkTheme.css.ts
 │   │   └─ index.ts
 │   │
 │   ├─ utils/                  # 공통 스타일 유틸 (sprinkles 등)
 │   │   ├─ sprinkles.css.ts    # responsive, spacing, color 등 통합 유틸
 │   │   └─ index.ts
 │   │
 │   └─ globals.css.ts          # reset, global style
 │
 ├─ components/                 # UI 컴포넌트
 │   ├─ Button/
 │   │   ├─ Button.tsx
 │   │   ├─ Button.css.ts
 │   │   └─ index.ts
 │   │
 │   ├─ Text/
 │   │   ├─ Text.tsx
 │   │   ├─ Text.css.ts
 │   │   └─ index.ts
 │   │
 │   └─ ...
 │
 └─ app/                        # 실제 앱 레벨
     ├─ pages/
     ├─ features/
     └─ ...
```
