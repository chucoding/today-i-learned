# cursor
프론트엔드 에디터 Cursor 사용법 정리

## 요금제

### Team
- 월 500개의 **프리미엄 요청 포함 사용량**이 제공됨
- 500을 초과하면 고급 모델 차단 (하드캡 적용된 경우)

## Agent Modes

### Agent
- 코드 작성할 때 주로 사용
- Windows에서는 실행할 때 보통 PowerShell을 사용

### Ask
- 중간 설계 질문시 사용
- 전체 아키텍처 재검토급 질문은 MAX Modes로 사용

### Plan
- 설계 작업

### Debug
- 디버깅 작업

## Context Modes

### Auto
- Cursor에서 제공하는 모델 라우터
- 작업 유형에 따라 모델 자동 선택(어떤 모델 사용하는지 공개되지 않음)
- 일반 질문 / 간단 수정 / 잔에러 처리에 적합
- 중간 설계 질문은 대부분 Auto로 충분

### MAX
* 선택한 모델을 "최대 성능 설정"으로 실행
* 더 큰 컨텍스트 사용
* 더 깊은 추론
* 토큰 사용량 증가
* 설계 확정, 아키텍처 판단에 적합

### Use Multiple Models(거의 안씀)
* 여러 모델이 병렬 실행 후 결과 비교
* best-of 전략
* 정확도 상승 가능
* 사용량 소모 매우 빠름
* 단일 모델만 활성화된 경우 의미 거의 없음

## Model

| 모델            | 특성      | 추천 상황      | 비용 |
| ------------- | ------- | ---------- | --- |
| Composer 1.5  | 빠르고 가벼움 | 단순 요약/상호작용 | 무료 |
| Sonnet 4.6    | 균형     | 일반 작업      | 1 Request |
| Opus 4.6      | 추론 강함   | 설계/아키텍처 판단 | 많이듬 |
| GPT-5.4 | 코드 특화   | 대량 코드 수정   |많이듬(MAX mode) |


## 최종 권장 운영 패턴

1. 설계 확정 → Opus + MAX
2. 대량 치환 → Codex
3. 잔수정/질문 → Auto
